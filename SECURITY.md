# Security and public-repository boundary

Report accidental publication of credentials, personal information, private
operations material, or other sensitive content to `info@ieset.org`.

Do not open a public issue containing the sensitive value itself. Include the
affected path or URL and a non-sensitive description. IESET will preserve
research provenance where possible while prioritising removal of exposed
private or security material.

For methodological errors, data corrections, and replication failures, use the
public process in [CONTRIBUTING.md](CONTRIBUTING.md).
