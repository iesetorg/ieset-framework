# PANEL Candidate Revival Audit - 2026-05-15

Scope: blocked or shallow PANEL-style candidates that can be revived with data already present in the local workspace. This audit intentionally does not edit YAML, position files, or run artifacts.

## Selection Principles

- Prefer candidates whose current blocker is stale because a local vintage has landed after the original run.
- Prefer candidates that can reuse existing runners or nearby specialist scripts with small data-wiring changes.
- Prioritize OECD PMR/PDB/STAN-compatible productivity panels, BIS financial-cycle panels, Eurostat energy/regional panels, IRENA learning-curve data, minimum-wage bite panels, WID inequality data, and Tax Foundation ITCI tax panels.
- Avoid candidates whose real blocker is still native microdata, metro data, or manual coding that is absent.
